Docker私有仓库的搭建
7月 28 2020

Docker私有仓库的搭建

开始搭建

  1. 下载registry镜像

    sudo docker pull registry

  2. 查看镜像是否下载成功

    sudo docker images

  3. 下载后以后启动该容器，可以将容器内的数据映射挂载在自己指定的目录上，这里以/opt/data/registry为宿主机存储的目录创建宿主机储存的目录

    mkdir -p /opt/data/registry

  4. 启动容器

    sudo docker run -d -p 5000:5000 -v /opt/data/registry:/var/lib/registry --name private_registry registry //启动容器

    -d : 让容器可以后台运行

    -p ：指定映射端口（前者是宿主机的端口号，后者是容器的端口号）

    -v ：数据挂载（前者是宿主机的目录，后者是容器的目录）

    –name : 为运行的容器命名

  5. 查看镜像是否启动成功

    sudo docker ps

测试本地私有仓库是否搭建成功

（此处ip为宿主机ip地址，端号为registry容器所映射的端号，例如ip地址为192.168.56.101，registry容器所映射的端号为5000）

  1. 拉取一个较小的镜像进行尝试

    Docker pull busybox

  2. 给一个已有的镜像打上标签

    Docker tag busybox ip:port/busybox

  3. 将这个tag镜像push到本地仓库

    Docker push ip:port/busybox

  4. 再删除掉所有关于busybox的镜像

    Docker rmi busybox

    Docker rmi ip:port/busybox

  5. 查看镜像是否删除成功

    Docker images

  6. 从本地私有仓库拉取busybox镜像

    Docker pull ip:port/busybox

  7. 查看是否拉取成功

    Docker images

假:设宿主机的ip地址为192.168.56.101

打标签命令 docker tag busybox 192.168.56.101:5000/busybox

将tag镜像push到私有仓库 docker push 192.168.56.101:5000/busybox

从本地私有仓库拉取镜像docker pull 192.168.56.101:5000/busybox

在搭建的过程中可能会出现的错误

Get https://192.168.0.180:5000/v1/_ping: http: server gave HTTP response to HTTPS client

解决办法

在”/etc/docker/“目录下，创建”daemon.json“文件。在文件中写入：
{ "insecure-registries":["192.168.0.180:5000"] }

保存退出后，重启docker。问题解决

查看本地私有仓库的命令

查看私有仓库镜像列表

curl -XGET http://192.168.56.101:5000/v2/_catalog

查看busybox镜像的标签列表

curl -XGET http://192.168.56.101:5000/v2/busybox/tags/list
