Redis的开发规范以及命令的基本使用
7月 22 2020

Redis的开发规范以及命令的基本使用

阿里云开发规范

一 键值设计

1.key设计

  • 可读性和管理性
  • 简洁性
  • 不要包含特殊字符

示例：以业务名 (或数据库名) 为前缀 (防止 key 冲突)，用冒号分隔，比如业务名: 表名:id，在保证语义的情况下可以适当简化控制key的长度

userManager : user: {uid} 可以简化为um:u:{uid}

2.value设计

  • 拒绝bigkey
  • 选择适合的数据类型
  • 通过设置过期时间来控制生命周期

示例：hmset user:1 name tom age 19 favor football

3.Redis 事务功能较弱，不建议过多使用

4.避免多个应用使用一个 Redis 实例，要根据不同的业务来进行拆分

5.推荐使用Redis连接池

其他规范请参考：Redis开发规范 https://www.infoq.cn/article/K7dB5AFKI9mr5Ugbs_px

命令的基本使用

Redis 官方也提供了在线的调试器，你也可以在里面敲入命令进行操作：http://try.redis.io/#run

字符串

设置和获取键值对

SET key value
GET key

是否存在键值对 EXISTS key

删除键值对 DEL key

批量设置键值对 MSET key1 value1 key2 value2

批量获取键值对 MGET key1 value1

给已存在键值对设置过期时间 EXPIRE key time

例: EXPIRE key 5 给名为key的键值对设置5秒后过期

设置键值对并且设置过期时间 SETEX key 5 value

计数

SET counter 100
INCR counter //返回101
INCRBY counter 100 //返回151

设置键值对并返回原值 GETSET key value

列表

向 list 左边插入数据 LPUSH list value

向 list 右边插入数据 RPUSH list value

从 list 中取出一定范围的元素 0 -1 表示全部 LRANGE list 0 -1

从 list 中取出指定下标的元素 LINDEX list 0

登出 list 左边第一个元素元素 LPOP list

登出 list 右边第一个元素元素 RPOP list

字典hash

创建hash并设置键值对 HSET books key value

获取字典中所有的键值对 HGETALL books （不推荐使用）

获取键值对 HGET books key

批量设置键值对 HMSET books key value key1 value1

集合set

插入元素 SADD books value

获取Set中所有元素 SMEMBERS books

查询某个value是否存在 SISMEMBER books value

获取集合长度 SCARD books

弹出一个元素 SPOP books

有序列表 zset

> ZADD books 9.0 "think in java"
> ZADD books 8.9 "java concurrency"
> ZADD books 8.6 "java cookbook"

> ZRANGE books 0 -1     # 按 score 排序列出，参数区间为排名范围（即下标范围）
1) "java cookbook"
2) "java concurrency"
3) "think in java"

> ZREVRANGE books 0 -1  # 按 score 逆序列出，参数区间为排名范围（即下标范围）
1) "think in java"
2) "java concurrency"
3) "java cookbook"

> ZCARD books           # 相当于 count()
(integer) 3

> ZSCORE books "java concurrency"   # 获取指定 value 的 score
"8.9000000000000004"                # 内部 score 使用 double 类型进行存储，所以存在小数点精度问题

> ZRANK books "java concurrency"    # 排名
(integer) 1

> ZRANGEBYSCORE books 0 8.91        # 根据分值区间遍历 zset
1) "java cookbook"
2) "java concurrency"

> ZRANGEBYSCORE books -inf 8.91 withscores  # 根据分值区间 (-∞, 8.91] 遍历 zset，同时返回分值。inf 代表 infinite，无穷大的意思。
1) "java cookbook"
2) "8.5999999999999996"
3) "java concurrency"
4) "8.9000000000000004"

> ZREM books "java concurrency"             # 删除 value
(integer) 1
> ZRANGE books 0 -1
1) "java cookbook"
2) "think in java"
